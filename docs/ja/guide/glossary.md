# 用語集

kaizen-spec 全体で使われる改善・看板用語のクイックリファレンスです。例とスキルのマッピングを含む完全な定義は[完全版リファレンス用語集](/ja/reference/kaizen-glossary)を参照してください。

---

## フロー用語

| 用語 | 日本語 | 一行の意味 |
|---|---|---|
| **Kanban** | 看板 | 看板 — 仕事を見える化し過負荷を制限するシステム |
| **WIP Limit** | 在製品限制 | 1カラムに同時に許可されるタスクの最大数 — 「始めるのを止め、終わらせることを始めよ」 |
| **Lead Time** | 交付周期 | `created_at` → `completed_at` — システム内の総時間（カードでは LT として表示） |
| **Cycle Time** | 週期時間 | `started_at` → `completed_at` — 実作業時間のみ（カードでは CT として表示） |
| **Takt Time** | タクト時間 | 需要を満たすために必要なデリバリー速度 |
| **Pull System** | 引き取り | エージェントがキャパシティの空きに応じてタスクを自己アサインする — 押し付けではなく引き取り |
| **Heijunka** | 平準化 | 負荷の平準化 — WIP制限がバースト→アイドルのパターンを防ぐ |
| **Bottleneck** | ボトルネック | 最も遅いカラム — まずそこにキャパシティを追加する |

## ムダ用語

| 用語 | 日本語 | 一行の意味 |
|---|---|---|
| **Muda** | 無駄 | ムダ — 価値を加えずにリソースを消費するあらゆる活動 |
| **Mura** | 斑 | ムラ — 不均一なフロー（対策：平準化） |

## 改善用語

| 用語 | 日本語 | 一行の意味 |
|---|---|---|
| **Kaizen** | 改善 | 改善 — 継続的な漸進的改善 |
| **Standard Work** | 標準作業 | 合意されたベースライン — `.kaizen/` ファイルが kaizen-spec の標準作業 |
| **PDCA** | 計画・実行・評価・改善 | 計画 → 実行 → 評価 → 改善 — TDD の赤→緑→リファクタリングに直接対応 |

## ボード上の表示

各かんばんカードには、`tasks.json` のタイムスタンプから計算された **CT**（サイクルタイム）と **LT**（リードタイム）が表示されます。カードにカーソルを合わせると詳細な内訳が確認できます。

「完了」カラムは `done_visible_max` 枚のカードまで表示されます（デフォルト：10枚）。古いカードは `tasks.json` にメトリクス用として残り、カラム下部にフェードしたアーカイブ・スタブとして要約されます。
