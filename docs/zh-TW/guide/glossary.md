# 詞彙表

kaizen-spec 全文所使用的改善與看板術語快速參考。如需含範例與技能對應的完整定義，請參閱[完整參考詞彙表](/reference/kaizen-glossary)。

---

## 流動術語

| 術語 | 日本語 | 一行說明 |
|---|---|---|
| **Kanban** | 看板 | 生產看板——使工作可視化並限制過載的系統 |
| **WIP Limit** | 在製品限制 | 某一欄位同時允許的最大任務數——「停止開始，開始完成」 |
| **Lead Time** | 交付周期 | `created_at` → `completed_at`——在系統中的總時間（卡片上顯示為 LT） |
| **Cycle Time** | 週期時間 | `started_at` → `completed_at`——僅計算主動工作時間（卡片上顯示為 CT） |
| **Takt Time** | タクト時間 | 為滿足需求所需的交付速率 |
| **Pull System** | 引き取り | Agents 在有空閒容量時自行認領任務——絕不被動推送 |
| **Heijunka** | 平準化 | 負載平準化——WIP 限制防止爆衝後停滯的模式 |
| **Bottleneck** | ボトルネック | 最慢的欄位——應優先在此增加容量 |

## 浪費術語

| 術語 | 日本語 | 一行說明 |
|---|---|---|
| **Muda** | 無駄 | 浪費——任何消耗資源卻不增加價值的活動 |
| **Mura** | 斑 | 不均衡——不規則的流動（解方：平準化） |
| **Muri** | 無理 | 過度負荷——將 agents 推超過容量（解方：WIP 限制） |

## 品質術語

| 術語 | 日本語 | 一行說明 |
|---|---|---|
| **Jidoka** | 自働化 | 自働化——機器偵測缺陷並停止；在軟體中 = TDD |
| **Andon** | 安燈 | 停止信號——阻塞卡片徽章 + WARN 日誌 + agent 停止工作 |
| **Poka-Yoke** | ポカヨケ | 防呆——階段門閘、schema 驗證、靜態型別 |
| **Genchi Genbutsu** | 現地現物 | 現地現物——直接讀取原始日誌，絕不依賴摘要 |
| **Gemba** | 現場 | 現場——實際的測試輸出、實際的 `.kaizen/` 狀態 |
| **5S** | 職場環境整備 | 職場整理——整理、整頓、清掃、清潔、躾 |

## 改善術語

| 術語 | 日本語 | 一行說明 |
|---|---|---|
| **Kaizen** | 改善 | 變更為更好——持續漸進式改善 |
| **Standard Work** | 標準作業 | 已協定的基準——`.kaizen/` 檔案是 kaizen-spec 的標準作業 |
| **PDCA** | 計画・実行・評価・改善 | 計劃 → 執行 → 查核 → 行動——直接對應 TDD 紅→綠→重構 |

## 看板上的資訊

每張看板卡片顯示從 `tasks.json` 的時間戳計算出的 **CT**（週期時間）與 **LT**（交付周期）。將游標移至卡片上可查看完整明細。

完成欄位上限顯示 `done_visible_max` 張可見卡片（預設 10 張）。較舊的卡片仍保留在 `tasks.json` 中用於指標計算，並以淡化的封存摘要顯示在欄位底部。
